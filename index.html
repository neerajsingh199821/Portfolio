    HubSpot to Eloqua Migration Project



    Project Objective

    Migrate all marketing assets and data from HubSpot to Eloqua for comprehensive campaign management, while retaining essential workflows, lead scoring, and contact segmentation.

    Timeline

    July 2023 - October 2023

    Project Overview

    This project aimed to transfer all crucial data and functionality from HubSpot to Eloqua, ensuring an efficient migration with minimized disruption. Key elements included contact data, email templates, automation, landing pages, and forms.

    Scope of Migration

    • Contacts Database & Lists: Exported and cleaned contact data by list in HubSpot, mapped fields in Eloqua, and verified correct data import. Contacts were then organized into shared lists for easy segmentation.
    • Lead Scoring: Documented HubSpot’s criteria and point assignments. Designed a similar, customized lead scoring model in Eloqua, enhancing the model for Eloqua’s environment to support better lead nurturing.
    • Email Content: Redesigned and recreated emails using Eloqua’s template options. Images were uploaded directly, while complex HTML files were migrated with custom coding.
    • Automation Campaigns: Recreated HubSpot’s workflows as multi-step campaigns in Eloqua, using decision and action elements for seamless automation.
    • Landing Pages and Forms: Migrated landing pages by recreating design elements and functionalities, utilizing Eloqua templates and custom coding for advanced customization.
    • Files and Images: Uploaded all gated content and images to Eloqua’s asset manager, ensuring no broken links or missing visuals in the final setup.

    Challenges Faced

    • Data Mapping Issues: Managed field discrepancies between HubSpot and Eloqua’s data structures.
    • Automation Differences: Adapted HubSpot workflows to Eloqua’s environment.
    • Content Formatting Adjustments: Redesigned emails and landing pages for visual consistency.

    Key Achievements

    • 100% Asset Migration with validated contact lists and custom lead scoring.
    • Provided thorough client training for smoother transitions.
    • Optimized Workflow Configurations for efficient automation.

    Outcomes & Impact

    • Enhanced Data Accuracy and Reporting Insights: Improved segmentation and analytics.
    • Seamless Transition: Zero disruptions in active campaigns, with an improved automation strategy and updated asset library.

    Calendly & Eloqua landing page Integration



    Project Objective

    The project aims to integrate a meeting scheduling feature into Eloqua landing pages using Calendly, enabling customers to schedule meetings with sales representatives based on their availability. The integration will involve accessing Keysight employee calendar availability to display only the available time slots.

    Timeline

    July 2023 - October 2023

    Goals

    • Enable customers to schedule meetings with sales representatives directly from Eloqua landing pages.
    • Ensure that only available time slots are displayed to customers, based on Keysight employee calendar availability.
    • Allow customization of scheduling options such as appointment duration and sales representative selection.
    • Implement confirmation emails and notifications for both customers and sales representatives upon successful meeting scheduling.
    • Implement integration of Microsoft Teams with Keysight employee calendars to retrieve meeting schedules.

    Individual Flow

    • Users open the branded landing page and navigate to the Calendly scheduling feature.
    • They select a pre-configured meeting type, choose a date and time, and enter essential details.
    • Upon confirmation, users receive an MS Teams invitation with complete meeting information.
    • Optionally, users can add agenda items or invite additional guests before finalizing.

    General Flow

    • Users open the link to the company-branded web app.
    • After selecting location preferences (e.g., country, state) or auto-detection, they choose an account manager for scheduling.
    • Users can view and select from available time slots with the chosen representative. If no preferred times are found, they can choose another manager.
    • Calendly displays customized meeting types set by the account manager. After selecting a time, users enter details to receive an MS Teams invitation.
    • Users can add an agenda or invite guests, then confirm the booking on the final page.

    Key Features

    • User-Centric Scheduling: Enabled users to select country, state, and preferred account managers.
    • Calendar Sync: Allowed account managers to sync their Keysight calendars with Calendly.
    • Meeting Details: Customized events displayed based on the account manager's setup, with scheduling details delivered to users via MS Teams.
    • Advanced Security: Integrated Zapier to secure data flow and manage user sessions efficiently.

    iFrame and Enable IDP on Eloqua Landing Page



    Objective

    Develop a secure Eloqua landing page with an embedded Domo link, accessible only to authenticated users through Single Sign-On (SSO) and email-based filtering.

    Timeline

    Feb 2024 - Apr 2024

    Overview:

    In this project, I designed and implemented an authentication-controlled iFrame integration on an Eloqua landing page. The solution ensures that only verified users can access embedded Domo content by authenticating against an Identity Provider (IDP) and applying access restrictions based on user emails.

    Solution Details:

      Authentication Setup:
      • Registered the application in Azure's App Registrations, allowing the page to perform SSO using the IDP. This step included setting up a client secret key, Application ID, and Tenant ID for secure access.
      Landing Page Creation in Eloqua:
      • Built the Eloqua landing page with an embedded iFrame showing a public Domo link.
      • Configured custom JavaScript to handle authentication logic and ensure a smooth user experience.
      Conditional Access Control:
      • Integrated a filtering mechanism based on email, stored securely in Eloqua, to validate access.
      • Verified user credentials from query parameters, ensuring iFrame visibility only to authorized email addresses.
      Enhanced User Experience:
      • Added a background blur effect that activates during authentication and removes upon successful login, enhancing security and UX.
      Dynamic pFilter Integration:
      • Implemented a URL modification feature, fetching pFilter data from query parameters and constructing the resulting link to display within the iFrame.
    This project demonstrated advanced use of Eloqua for secure content delivery, seamless SSO integration, and enhanced user experience tailored to enterprise needs.

    Key Skills Utilized:

    • Eloqua, IDP (Azure), SSO, JavaScript, iFrame Integration, Domo, API, and Authentication Management.
